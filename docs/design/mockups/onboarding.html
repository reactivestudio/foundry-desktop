Флоу: шесть шагов, по решению на экран. Порядок — это зависимости, а не важность: Agent → Extensions (плагин + CLI, оба обязательные) → Settings → Permissions → Ready. Правила бессмысленны до выбора агента, движку нечего гонять до правил, настройки спрашиваются когда уже понятно, что они настраивают. «Agent» — два честных мира. В свежей системе на всех трёх карточках Install (ультрамарин — это действие экрана), а требование написано до клика («Pro / Max plan or API key»): человек узнаёт цену входа не после установки. У найденного агента кнопки нет — паспорт отвечает фактами («✓ v2.1.4 · signed in»), и клик по карточке и есть выбор. Install симулируется: Installing… → карточка перещёлкивается → автопереход к правилам — траектория настоящего инсталлера. Успех встаёт РОВНО в слот кнопки (ревью 2026-07-21): «✓ vX · signed in» несёт margin-top:auto и падает к нижней кромке паспорта, как сама кнопка, — Install → Installing… → ✓ идут в одной точке, а не прыгают в паспорт; вторичный факт (план/путь) занимает место требования под именем. Строка детекта («scanned PATH…») жила под карточками и снята по ревью — проверкой изъятием: состояние системы уже сказано кнопками и фактами карточек, строка была дублем. Карточка — флаговая вёрстка (ревью того же дня): лого стояло по центру — а у карточки с текстом ось одна, левая; теперь тайл, имя, факты и Install стоят на одной оси (вендор прожил колонку → строку имени справа (№7, ломал имя на две строки) → снят вовсе (№8) → вернулся НАДСТРОЧНИКОМ над именем (№9, «вернуть, но мельче и сверху»): над именем ширины хватает, переносов нет; кегль 8 капсом, все шрифты паспорта ужаты на ступень, чтобы всё влезло), ряды выровнены сквозь карточки фиксированной высотой тайла, зазор сетки 24 поднят над паддингом 16 — внутреннее СТРОГО меньше внешнего, равенство уже слипание. Части Foundry прошли три формы: «Rules for Claude»/«Engine» (имена врали: не правила, а плагин; не движок, а CLI), потом честные отдельные «Plugin»/«CLI», теперь — один экран Extensions (ревью: решение-то одно — поставить обе части; и добавка Agent → «Agent extensions» по смыслу). Экран говорит языком «Agent»: та же композиция — рой отпущен завесой, заголовок и лид по центру, решение экрана у нижней кромки, — и две квадратные карточки той же геометрии, что у агентов; дальше мастер ведёт только когда установлена пара, после первой карточки экран ждёт вторую. (По дороге была форма «одиночная горизонтальная карточка-строка» — умерла вместе с отдельными экранами.) Знаки частей — пиктограммы набора мастера, белый штрих 1.25 на сетке 24, БЕЗ подложки (ревью №5: «подобрать с умом, без фона» — тайл-заливка и «Восход» с карточки сняты: логотип продукта уже стоит в вордмарке, на карточке нужен знак роли, не бренд): у плагина — пазл, часть, встающая в вырез Claude Code; у CLI — терминальные шеврон и подчерк (путь знаков: серый моноширинный ›_ → орб → двухцветный ультрамарин-циан → «Восход»+штрих 5 — все сняты по ревью). Пустой тайл-бокс ужат до знака: 48-й раздувал зазор знак→вендор до 24 против паддинга 16 — слипание лестницы; теперь визуальные 12, как у агентов. Одна строка факта до установки («7 skills · 4 agents · 5 commands» / «worktrees · git · stage runner»), Install в карточке. Строка контракта под сеткой («plugin goes to ~/.claude… · CLI — 8 MB binary…») снята (ревью №8): куда лягут файлы — забота установщика, не витрины выбора; проверка изъятием — смысл экрана цел. План-дифф по ~/.claude снят ещё раньше по той же логике. «Settings» — компактная плашка в языке экранов-предложений (ревью: разворот на всю ширину занимал слишком много места): пять настроек — строками в той же плашке .setpanel, что Permissions и резюме «Готово» (ревью №5: «Settings по аналогии с Ready»; путь формы: плашка с тумблерами → 154×154 в 4+1 → ряд из пяти 124×124 → два ряда 3+2 → снова плашка — три экрана подряд говорят одним компонентом, отдельная сетка настроек была вторым языком). Строка плашки: имя + объяснение слева (пара, её держит близость — зазор 2, между строками воздух в разы больше), тумблер справа; пиктограмм в строках нет — их нет ни у Permissions, ни у резюме. Имена ужимались трижды ещё в карточной форме («Human review before merge» → «Review before merge» → «Merge review», «Token in Keychain» → «Keychain») и короткими остались. Пиктограммный набор мастера (сетка 24, штрих 1.25, оптические габариты ~14–16 единиц по большей стороне, размер меряется глазом, а не боксом; путь: белый штрих 4.5 → двухцветные жирные → белый 4 → Lucide 1.5 → финальная геометрия) с настроек ушёл вместе с карточками и живёт теперь на «Extensions». Мишень нажатия — вся строка по закону Фитса, тумблер лишь показывает состояние (ревью 2026-07-21: у него `pointer-events:none` и нет своего обработчика — клик всегда ловит строка; прежний отдельный обработчик со `stopPropagation` снят). Тумблеру добавлен минималистичный объём по СВЕТУ (не гало, его на проекте отвергали): канавка утоплена внутренней тенью сверху + светлая нижняя кромка — и в «вкл» ТОЖЕ (включённое меняет только цвет, канавка всегда вдавлена), кружок приподнят мягкой тенью и верхним градиентом. Кружок — простой круг, едет ровным транзишеном без деформации (капля-эластик и пружинный овершут пробовались и сняты по просьбе — «оставим просто круг»). Continue съездил в сетку CTA-карточкой с поверхностью primary (ревью №2: «отдельная кнопка съедает пространство»), вернулся кнопкой (ревью №3: «карточка смотрится плохо») и кнопкой остался под плашкой — без хоткея-подсказки. Прежние формы — разворот, плашка-таблица, плашка с объяснениями, колонка строк, прямоугольники 208×132, квадраты 160 в два ряда 3+2 с белыми пиктограммами на тайлах — сняты по ревью. Лид экрана («Five defaults, flip them any time») и строка про папки проектов не прошли редактуру по Ильяхову — очевидности; сняты, у Settings остались заголовок и сетка карточек с CTA внутри. Экран «Link» выпилен (ревью: непонятен и незачем) — рукопожатие desktop↔claude уедет в фоновые проверки настоящего приложения, мастеру хвастаться им рано. Экран «Project» снят: подключение проекта — работа настоящего окна; мастер заканчивается на «Ready» со сводкой фактов, и пустое состояние главного окна «Подключить проект…» подхватывает ровно там, где мастер отпустил. Раунд якорей (самопроверка по бюро): заголовки экранов-предложений прыгали по вертикали, потому что высоту нижней зоны диктовал контент, — теперь она зажата фикс-доком, и заголовок с лидом стоят на одном месте у кромки орба (замер по кадрам: 972/960/972 px, разница — длина строки, не позиция). Вертикаль прошла три формы. Три высоты дока (220/436/344) — заголовок прыгал на каждой границе («отступы скачут»). Единый док 436 — заголовки коротких экранов уехали наверх от своего контента и повисли в пустоте (ревью 2026-07-20 №2: «Settings почему так высоко уехал от карточек? Permissions тоже, Ready тоже»). Итог: высоту диктует контент, а причёсан РИТМ — от лида (или заголовка без лида, у него .solo с тем же полем) до контента везде ровно 40; нижняя кромка — общий якорь, заголовок сидит у своего решения, а не у чужого роста. От раунда единого дока остался тёмный ореол цветом фона под заголовком и лидом (две тени, 10px и 4px): как эффект не виден, контраст на частицах возвращает; приветствие не тронуто. Требования на карточках («Pro / Max plan or API key») были покрашены цветом недоступности — а это информация, не запрет: перекрашены в третичный, цвет недоступности остался только настоящим disabled. Лид Settings повторял слово «Settings» дважды — переписан, а следующим ревью снят целиком. Раунд слияния и предметности. Новый экран Permissions — в том же языке предложений: плашка с двумя правами (Notifications — «стейдж кончился или ждёт ревью», Accessibility — «глобальный ⌥ Space и панель чёлки»), Allow — ультрамарин действия экрана; после клика кнопка превращается в зелёный факт «✓ Granted», оба права выданы — мастер сам ведёт дальше. Доступ к папкам честно отложен строкой под плашкой: macOS спросит при подключении проекта. «Ready» — цепочка сборки вместо списка фактов: не отчёт о вещи, а сама вещь — конвейер слева направо, Claude Code пишет → плагин учит → foundry гоняет → чёлка показывает; узлы — те же тайлы, что на карточках (словарь один), имена — ряд сквозь модули, факты — моноширинный ряд под ними, между тайлами нить. Ярлыки фактов теперь и есть имена узлов. Рой на «Ready» наезжал на тайлы цепочки — добавлено третье состояние завесы veil-ready: от заголовка вниз частицы въезжают в градиент к bg.base (прозрачно до 54% высоты, солид к 74%) и гаснут, не доехав до узлов, — подъезжают, а не наезжают; приветствие не тронуто, там завесы по-прежнему нет. Тайлы цепочки были полупрозрачными и просвечивали фоном (ревью) — теперь у них чёткая поверхность: тон вендора поверх той же плиты, что у карточек, с bg.base в подложке и мягкой тенью. И этого не хватило: плиты с подобранными фонами всё равно смотрелись разнобоем, композиция — монотонной (ревью). Была и формула сборки (✳ + восход + ›_ + чёлка = Foundry AI) — «иллюстрация формулы — отстой», и был голый титул с вордмарком кеглем 48 — «ушёл в плохую сторону»: имя без содержания. Финальная форма «Ready» — резюме сетапа (ревью 2026-07-20: «там нужно резюме»): та же плашка и те же строки, что у Permissions, — четыре шага мастера, четыре строки-итога (Agent — Claude Code v2.1.4 · Max plan; Extensions — плагин и foundry с версиями; Settings — что включено; Permissions — что выдано), справа у каждой зелёный факт-галка. Экран не выдумывает новой формы — закрывает мастер тем же словарём, которым тот говорил всю дорогу. Лид-поздравление в две строки по формуле остальных экранов («That’s the whole setup / Foundry takes it from here») — итог и передача эстафеты, без «congratulations». Плашка сидит прямо на частицах — под ней многослойная тень-посадка (shadow-soft + два слоя всё шире и мягче книзу): не объём, а притемнение роя под кромкой. Кнопка финала примерялась широкой «в ширину плашки» — не оправдала себя (ревью №3): вернулась обычная, как на приветствии, со словом действия «Start working» и без хоткея-подсказки — финалу хватает одного слова. Финальный поворот (ревью 2026-07-21): завесы сняты со всех экранов вовсе — солид у нижней кромки делал фон разным сверху (частицы) и снизу (глухая заливка). Рой живёт во весь кадр, а отделение контента несёт общая парящая тень --shadow-float: четыре слоя 96/210/400/700px, альфы 0.96/0.80/0.54/0.30 — почти непрозрачное чёрное ядро (96–210px), провал вдвое шире градиентно затемняет рой на 30–50px вокруг кромки, плюс мягкий хвост во все стороны. Путь: 4 слоя до 96px → 11 до 280 → 14 до 680 → 15 до 800 — и седьмое ревью («опять не делаешь: на Settings, Permissions и Ready тени нет») открыло потолок блюра: у одиночной плашки 340×150 гиганты 680/800 растворялись — блюр много больше тела размазывает альфу по своей площади, и у кромки вклад стремится к нулю (замер: Permissions гасил дымку лишь в 30px; дотягивается box-shadow примерно на полблюра). Энергию собрали в среднем поле до 456, дальние гауссианы сложили стопкой — одиночный слой такой ширины у малого тела плотного гало не даёт физически, двойной гасит дымку вдвое глубже и дальше (стенд); теперь градиент читается через все 350px до соседа. Дёрганность перехода лечили трижды: сначала 18 лёгких слоёв → 7 плотных (число слоёв), потом промоушен экрана в слой (will-change) — и всё равно дёргалось (ревью №9 «наконец-то исправь»). Настоящая причина оказалась в РАДИУСЕ, а не числе: краска box-shadow растёт как блюр², и на смене экрана входящий красится за один кадр целиком (display none→flex) поверх WebGL-роя — стопка 456px роняла кадры. Потолок опущен 456→240: краска ~3.6× дешевле на большом слое, спайк входит в бюджет; сила не потеряна — near-field ПЛОТНЕЕ (0.60 на 20px), а 240 на теле 184–340 near-оптимум (блюр ≈1.3× тела). will-change оставлен. Плотный near-field и оказался «чёрным кантом» (ревью №10, дважды: «у карточек будто чёрный бордер, когда рой заходит за них»): сумма высоких альф ближних слоёв (0.60+0.56+…) давала у самой кромки композит ~96% чёрного — на почти-чёрном фоне невидимый, но против яркого роя это глухая тёмная линия по контуру. Тугой ближний слой убран, вся лестница разрежена до четырёх мягких слоёв 44→248px, альфы 0.28→0.09: у кромки плавное затемнение вместо солид-обода (замер поперёк верхней кромки: было (5,4,8) near-чёрное, стало плавный спад (10,8,15) к фону — обода нет). Глубину теперь несёт размах поля, а не альфа у кромки; потолок 248 держит дешёвую краску перехода. Но альфы 0.28→0.09 оказались перелётом (ревью №11: «тени под карточками вообще пропали»): на почти-чёрном фоне тень видна ТОЛЬКО как притемнение дымки роя, а такие малые альфы, размазанные большими радиусами, дымку почти не гасят. Кант давал не вес, а малый радиус ближнего слоя — значит вес возвращаем, держа минимальный блюр большим: финал 4 слоя 56/130/224/320px, альфы 0.34/0.42/0.30/0.18, максимум веса в среднем поле (130px, 0.42), где реально стоит рой; кромка при этом остаётся градиентом (замер: самое тёмное у кромки (8,6,13) = уровень bg.base, не под-фон; тело плиты (22,20,30) светлее — карточка парит над затемнённым полем, линии-обода нет). Правило: сила тени — альфа в среднем поле; отсутствие канта — большой минимальный блюр. Тень живёт не на теле, а на ::before-подложке с z-index −1: box-shadow элемента рисуется в его слое и красил кромку соседней карточки — подложки же собираются под телами всего экрана, каждая карточка отбрасывает одинаковую тень и ни одна не темнит другую. Заголовки, лиды, кнопки и пагинация подняты слоем выше теней — сотни пикселей мягкого края не смеют их притемнять. Тела карточек при этом непрозрачные: рой бежит под ними, а не сквозь. Шов у кромки закрыт (ревью №9: «у роя за карточкой 1px прозрачный бордер, тени там нет»): тело карточки залито по padding-box (прозрачный бордер 2px краски не несёт), а ::before-подложка была пустой — сквозь кольцо просвечивал рой. Поле лейбла ровное не по зазору, а по площади. Глаз видит литеры, CSS отмеряет от бокса глифа, и между ними лежит то, что в бокс входит, но не красится: боковые полусапожки, разрядка после последней литеры, полулиддинг. Но и ровный зазор ещё не ровное поле: зазор — это минимум пустоты, а глаз считает площадь. «I» — прямая стойка, её поле справа читается ровной полосой, минимум равен среднему; «A» — треугольник, её поле слева и сверху это клин, и при том же минимуме пустоты там в полтора раза больше. Поэтому aiprobe.py меряет и min, и mean, а поля подобраны по среднему: «I» справа держит опору, слева и сверху поджато под клин «A», чтобы средние сошлись. Поверх этого расчёта бока раздвинуты на +2px решением — знаку нужен воздух, и это правка глазом поверх замера, а не вместо него: вертикаль осталась на расчётных полях, сдвинулись только бока, симметрично. Ровняется то, что судит глаз, а не то, что легче померить.

Смягчает её то, что янтарь уведён на 9° в оранжевое, и то, что пересечься им негде: лейбл живёт только на приветствии, где предупреждений нет, а жёлтые предупреждения — на «Компонентах», где нет лейбла. Угол кнопки — тот же гиперэллипс, что у иконки, и это замер, а не подбор. У CSS есть corner-shape, и его параметр — не кривизна, а показатель степени: superellipse(k) строит |x|ⁿ+|y|ⁿ=1 при n = 2ⁿ⁾ᵏ, то есть round = n 2, а squircle = superellipse(2) = ровно n 4. Угол иконки в этом репозитории уже был замерен по референсу incident.io и оказался n 4 при L = 0.4042 короткой стороны — значит кнопке ничего не надо подгонять, у неё и у иконки один и тот же угол по построению: border-radius: 18px; corner-shape: squircle, где 18 ≈ 0.4042 × 44. Тень под кнопкой — мягкая многослойная, без иллюзий о её громкости. Канон 09 § («на почти чёрном фоне тень невидима; границу задаёт свет») подтверждён стендом: я спорил с ним трижды — цитата вместо замера, WCAG-контраст 1.002:1 (чужая метрика), «7–9 ступеней из 13» (наличие вместо видимости) — и все три раза неверно; синий световой пролив как обход тоже отвергнут глазом. Состояния. На hover тело не двигается и тень не дышит: сдвиг дёргает глаз, а тень здесь отделяет кромку, не изображает высоту. Отвечает только заливка, и она отвечает вглубь, а не к свету: hover — −3.5% светлоты и +10° тона в пурпур (подбор глазом по вилке: 14° — событие, 6° — не видно, 10° — намёк), pressed — −8.5% той же дорогой. Светлеющий hover (канонные +7%) на почти чёрном кричал: единственная CTA и так самое яркое пятно интерфейса, прибавлять ей света некуда. Затемнение с уходом тона читается как отклик, не меняя громкости кнопки, — это отступление от буквы 08 § (±6–8% светлоты), названное в долге. Цвет ездит по кривой cubic-bezier(0.2,0,0,1) — почти вертикальный старт и длинный мягкий доезд, как гаснут переходные процессы в природе, — и асимметрично: вход в hover за 150ms, рука уже на кнопке и отклик обязан успеть за ней; выход назад к ультрамарину — за 450ms: внимание уже ушло, и цвет остывает, а не выключается. ⌘⏎ нарисован клавишей, со своей кромкой и своим бликом, потому что это и есть клавиша.

Подпись «1 из 6 · приветствие» ушла следом — она пересказывала словами то, что точки уже показали, и вдобавок называла экран, на котором пользователь стоит и так. Под кнопкой прибавлен воздух: между единственным решением экрана и служебной навигацией зазор обязан быть больше, чем между строками текста, иначе точки читаются частью кнопочной группы. Точки кликабельны — из этого следует два обязательства. Hover: точка плавно подрастает и наливается ультрамарином — обещает ровно то, чем станет после клика; активная не откликается, наведение на неё ничего не обещает. Мишень шире рисунка: канон это уже требовал — hit.min = 24×24 (13 §10, закон Фитса: время прицеливания растёт с уменьшением цели), и точки 6×6 его нарушали. Рисунок остаётся 6px, хит-зона каждой — 22×24: ряд при 24 стоял рыхло (просвет 18px), точки прижаты на 2px против буквы hit.min по ширине — отступление названо в долге. Зазоры выровнены по просветам, а не по слотам: капля шире точки на 12px, и раньше просвет вокруг неё был 12 против 18 у соседей — глаз это читал как неровный ряд; теперь слот активной шире остальных ровно на эти 12 (22→34), просвет между рисунками всюду 16px, а слоты дышат той же кривой, что капля, — соседи расступаются под неё. Активную рисует не сама точка, а полоска-индикатор, и при переходе она перетекает каплей: сперва тянется мостиком до цели, на середине мост во всю длину, дальше хвост отпускает и масса собирается в новой точке (460ms; первая половина резче, сбор мягче — у капли инерция, а не равномерный ход).   • Аспект. Канвас окна не квадратный, а в лоадере коррекции нет — она не была нужна: proj.x /= uAspect, иначе орб растянуло бы в эллипс.
  • Позиция. uCenter в NDC. Раньше орб переезжал между шагами за верхнюю кромку; теперь анимация на всех экранах одна — эталон приветствия, и рабочие шаги отличаются только контентом поверх (завесы сняты вовсе — отделение несёт тень).   • Разлёт — «как звёзды». Хореография из двух частей. Часть 1 (0–8% фазы, ~190ms): рой, не переставая жить своим морфингом, быстро сжимается к ядру до ×0.58 (ease-in-out) — замах. Часть 2 (8–100%, ~2.2s): из нижней точки — долгое расширение за границы экрана, кривая x²·(1.4−0.4x) с силой масштаба ×14: большая часть частиц скрывается за кадром уже к середине хода, остальное — медленный разгон остатка. Траектории — не прямые, а мягкие дуги: крутка вокруг центра с ростом угла как x^0.6 — изгиб набирается в начале полёта, пока частица ещё в кадре (линейный рост прятал всю дугу за кадром и был невидим); максимум 0.55–0.80 rad, направление одно на весь рой — деликатный вихрь. След повторяет дугу сам собой, потому что узлы считаются той же формулой. Обе кривые встречаются с нулевой скоростью — миг зависания в нижней точке читается замахом, а не швом. Поверх выброса — волна стартов 0–6% (перекат, а не строй) и кинематографичный наезд камеры (+40% uFit по кривой рывка — кадр втягивается в прыжок). Масштаб растёт по экранным осям (z не трогается, чтобы не простреливать камеру); приосевые частицы, которым чистый масштаб не прибавляет радиуса, выметает боковой снос. Личный разброс силы умеренный (×0.75–1.25, струи по направлению ×0.85–1.15): рой рвётся мягкими снопами, но летит ровно. Морфинг роя не прекращается во время сжатия — живёт до самого выброса; лишь в момент рывка внутренняя жизнь замораживается (t фиксируется на uT0), и частицы летят по гладким прямым, а не вибрируют на лету, как мухи. Гашение — по близости к камере, а не по времени: частица вылетает за кромку яркой, не умирая в кадре. Световые следы живут только у расширения (пандус 10–22% фазы) и дальше не гаснут; проявляются длиной, а не яркостью (до того сегменты вырождены — тёмная линия поверх частиц не возникает ни в одной фазе). Устройство следа: GL_LINES по узлам фазы (8 сегментов, шаг 0.015, личный разброс длины ±40%), резкая основная линия с затуханием по длине (степень 2.6), пол яркости хвоста — ровно bg.base (след растворяется в фоне, не оставляя тёмной царапины). Блюр хвоста — две тусклые копии (×0.30 яркости, гейт на хвостовую часть), сдвинутые перпендикулярно экранному направлению следа с ростом сдвига к хвосту: голова резкая, кончик мягко размыт. Яркая «оперёнка» на всю длину отвергнута — распушала след. След частицы у камеры гасится целиком (z в разлёте не меняется, оба конца отрезка решают одинаково; клип одного конца гнул бы линию к центру экрана). Блендинга нет, линии опаковые, глубину не пишут — семьи не смешиваются. Вспышка отпускания (+22%) подсвечивает момент рывка. Весь ход — 2370ms, окно передаёт сцену главному на 55%. Отвергнуты: «долгая волна стартов + короткий полёт» (рой испарялся, а не взрывался), «фейерверк: импульс + тормозной дрейф» (рвал рой рано, хвост пустел), длинные кометы отдельного акта разгона (нелепо), живой шум во время полёта (частицы разлетались «как мухи»).
