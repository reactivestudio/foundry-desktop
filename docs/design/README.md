# Design — дизайн-канон foundry-desktop

Серия документов о том, как выглядит и ощущается интерфейс foundry-desktop.
Свод школы бюро Горбунова (польза, УВЛ, якорные объекты, «Типографика и
вёрстка»), Ильи Бирмана (правило внутреннего и внешнего, модальность,
интерфейсные тексты), Эдварда Тафти (data-ink, small multiples, sparklines)
и классики взаимодействия (Фиттс, Хик, гештальт, эвристики Нильсена) —
привязанный к нашим экранам и палитре.

Серия — про **дизайн**: композицию, цвет, типографику, поведение. Код,
SwiftUI и механика macOS (Liquid Glass, окна, меню) — в соседней серии
[practices](../practices/README.md), прежде всего
[05-ui-design.md](../practices/05-ui-design.md); противоречий между сериями
быть не должно — при конфликте правится дизайн-серия в том же PR.

## Рамка (решено, не пересматриваем)

- **Только тёмная тема.** Светлая не проектируется вообще.
- **Палитра**: почти чёрный `#07060B`, ультрамарин `#2F5CFF`, пурпур
  `#8B5CF6`, маджента `#D65CFF`, циан `#58C7FF` как «точка света».
  Мотив — орб/«свет в темноте», концентрические круги. Ember отвергнут.
- **Аудитория** — один разработчик-эксперт; плотность десктопная,
  инструмент про чтение текста, кода и диффов.
- **Референсы поведения** — GitHub (ревью), Linear/Things (списки),
  Xcode/Terminal (лог).
- **Домен** — 8 стадий CRISPY: Questions → Research → Design → Structure →
  Plan → Worktree → Implement → PR; статусы change:
  backlog → in-progress → done/declined. Примеры в главах используют эти
  имена, других наборов стадий не существует.
- **Язык — только русский** (доменные термины — латиницей по словарю
  [05-text.md](05-text.md)); строки не проектируются под перевод.
- **Экран ≠ экспорт**: тёмная тема — для экрана; печать/PDF/шаринг наружу —
  светлая печатная палитра → [06](06-color.md).

## Сквозные принципы (действуют всегда)

1. **Дизайн — это польза** (Горбунов). Экран начинается с задачи, а не
   с картинки; красота — следствие точности → [01](01-principles.md).
2. **УВЛ — убрать всё лишнее.** Каждый элемент оплачивает своё место
   ответом «что сломается, если убрать» → [01](01-principles.md).
3. **Правило внутреннего и внешнего** (Бирман). Внутри ≤ снаружи; группы
   разделяет воздух, а не линейки → [03](03-spacing.md).
4. **Якорные объекты** (Горбунов). Экран держится на 1–3 якорях; иерархия
   читается прищуриванием → [02](02-composition.md).
5. **Каждый пиксель информирует** (Тафти). Data-ink максимален: без зебр,
   лишних сеток, декоративных градиентов и теней на данных →
   [10](10-tables.md), [11](11-dataviz.md).
6. **Цвет — дефицит.** Одна гамма, один акцент на зону, один glow в поле
   зрения; семантика цвета дублируется формой → [06](06-color.md).
7. **Никаких сырых значений.** Только токены: цвет, отступ, радиус,
   кегль, длительность. [13-tokens.md](13-tokens.md) — единственный
   источник значений; числа в остальных главах — иллюстрации, при
   расхождении прав токен-файл.
8. **Тёмная тема проектируется, не инвертируется.** Elevation — светлотой,
   текст — альфа-белым, акценты — осветлёнными → [06](06-color.md).
9. **Отклик мгновенен.** <100 мс на взаимодействие, честный прогресс,
   undo вместо «Вы уверены?» → [07](07-interaction.md), [01](01-principles.md).
10. **Платформа — macOS.** Системные паттерны и Liquid Glass не имитируем
    и не ломаем → [practices/05](../practices/05-ui-design.md).

## Как пользоваться

- **Рисуешь макет** — открыты [13-tokens.md](13-tokens.md) и глава темы;
  перед «готово» — прогон [14-checklist.md](14-checklist.md).
- **Ревьюишь макет/вёрстку** — идёшь по чек-листу главы, затем по сводному.
- Правило встретилось дважды в разных главах — каноничная формулировка там,
  куда ведёт ссылка; главы не пересказывают друг друга.
