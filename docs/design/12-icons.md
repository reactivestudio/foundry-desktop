# 12 · Иконки

> Серия design · [оглавление](README.md)

Эта глава — правила работы с иконками: когда иконка уместна, как жить с
SF Symbols, какие метафоры закреплены за доменными понятиями foundry-desktop,
размеры, статусы, кастомные символы, орб и анимация. Открывать при добавлении
любой иконки в UI и при ревью тулбаров/сайдбаров/списков. Цветовая семантика —
в [06-color.md](06-color.md), контролы вокруг иконок — в
[08-controls.md](08-controls.md).

## 1. Роль иконки

### 1.1 Иконка — ускоритель, не украшение

**Императив: иконка ставится, только если ускоряет распознавание уже знакомого;
декоративные иконки — вычеркнуть.** Почему: иконка — это сигнал; иконка «для
красоты» — шум, снижающий скорость чтения всего ряда
([01-principles.md](01-principles.md)). Как применить: у каждой иконки на
экране должен быть ответ на вопрос «что пользователь узнаёт быстрее благодаря
ей?». Нет ответа — убрать.

```
ПЛОХО:  ✦ Analytics   📁 Projects   ⚙ Settings — иконки-обои у каждого пункта, ряд пестрит
ХОРОШО: иконки только там, где по ним реально попадают глазами/мышью (сайдбар, тулбар)
```

### 1.2 Иконка без подписи — только вошедшие в язык

Нильсен: универсально понятных иконок — единицы; Бирман: иконка почти всегда
неоднозначна, пока рядом не побывал текст. Без подписи допустимы только
метафоры, вошедшие в язык интерфейсов:

| Можно без подписи | Нельзя без подписи |
|---|---|
| лупа (поиск), шестерёнка (настройки), ✕ (закрыть), + (создать), корзина (удалить), стрелки навигации, ··· (ещё) | «стадия», «луп», «артефакт», «request changes», «снапшот» — любые доменные понятия |

**Императив: доменная иконка живёт с подписью, пока не выучена, и всегда — с
тултипом.** У нас аудитория — один владелец, он выучит быстро, но правило
дешёвое: тултип обязателен у любой иконки-кнопки без текста (см.
[07-interaction.md](07-interaction.md) §12.2).

### 1.3 Текст + иконка: что дефолт

- **Меню** (меню-бар, контекстные) — всегда текст; иконки в пунктах меню macOS
  не ставит почти нигде, и мы не ставим (кроме статусных глифов-состояний).
- **Тулбар macOS** — конвенция платформы: иконка + тултип (+ подпись при
  режиме «Icon and Text», который решает система). Наши тулбарные действия —
  SF Symbol + тултип с шорткатом.
- **Кнопки в контенте** (approve, request changes) — текст обязательно; иконка
  опциональна как усилитель («✓ Принять»). Текст — носитель смысла, иконка —
  ускоритель повторного попадания.
- **Сайдбар** — иконка + текст (стандарт macOS sidebar). Схлопнутый сайдбар —
  единственный легальный случай «иконка одна», и тогда тултипы обязательны.

## 2. SF Symbols как система

SF Symbols — не набор картинок, а продолжение шрифта San Francisco: символы
спроектированы под совместную жизнь с текстом.

### 2.1 Что даёт система (и что не ломать руками)

- **Baseline-привязка**: символ сидит на базовой линии текста, как буква;
  не выравнивать иконку с текстом вручную по центру строки — сломается.
- **Кегль и вес наследуются от текста**: символ масштабируется с font size и
  подбирает толщину линии под вес шрифта. **Императив: кегль иконки = кегль
  соседнего текста, вес иконки = вес текста.** Regular-текст — Regular-символ,
  Semibold-заголовок — Semibold-символ; несовпадение веса читается как грязь.
- **Три масштаба** (small / medium / large) — не размер, а пропорция символа
  относительно того же кегля: medium — дефолт; small — плотные списки и бейджи;
  large — когда символ стоит один и должен весить больше текста (пустые
  состояния, крупные кнопки).
- **Оптические выравнивания уже внутри**: у каждого символа свои поля, чтобы
  круг, квадрат и стрелка казались одноразмерными. Поэтому не подгонять
  символы по габаритному прямоугольнику — они специально не совпадают.

### 2.2 Рендеры (rendering modes) — что используем в тёмной теме

| Режим | Что это | Используем? |
|---|---|---|
| Monochrome | один цвет целиком | **да — дефолт**: alpha-белый текстовых уровней |
| Hierarchical | один цвет, слои с разной непрозрачностью | **да** — для составных символов (значок с бейджем): вторичный слой сам притухает; в тёмной теме даёт глубину без второго цвета |
| Palette | 2–3 явных цвета по слоям | точечно: статусы, где нужен цвет+нейтраль (например, глиф + акцентная точка) |
| Multicolor | фиксированные «фирменные» цвета Apple | **нет** — пёстрые заливки разваливают тёмную палитру |

**Императив: monochrome — дефолт, hierarchical — для сложных символов,
multicolor — запрещён.** Почему: наша тема живёт на узкой палитре
(#07060B + ультрамарин/пурпур/циан), многоцветные глифы вносят чужие цвета.

### 2.3 Variable color и fill-вариант

- Многие символы имеют пары `*` / `*.fill` и variable color (частичная
  заливка слоёв по проценту). Конвенция macOS (и наша): **outline в покое,
  filled в выделенном/активном состоянии** — выбранный пункт сайдбара
  переключается на `.fill`. Это форма-дубль состояния, не только цвет.
- Variable color — для степени: громкость, прогресс, «заполненность лупа
  итерациями» (см. §9 про анимацию).

## 3. Метафоры доменных понятий foundry-desktop

Закреплённый словарь. **Императив: одно понятие — один символ, везде.**
Синонимия иконок (сегодня change — молния, завтра — карандаш) уничтожает
выученность.

| Понятие | SF Symbol | Обоснование |
|---|---|---|
| Проект | `folder` / `folder.fill` (выбран) | якорная метафора «контейнер работ»; узнаётся мгновенно, не конкурирует с доменными |
| Change | `arrow.triangle.branch` | ветвление — «единица изменения» с git-природой; отличим от папки и документа |
| Стадия | `circle` в вариантах: `circle.dotted` (не начата) → `circle.lefthalf.filled` (в работе) → `checkmark.circle.fill` (принята) | круг = узел пайплайна; заполненность = прогресс; согласуется с орб-мотивом (круги) |
| Пайплайн | `point.3.connected.trianglepath.dotted` или `arrow.right.circle` цепочкой | связанные узлы; в шапке канбана достаточно самих колонок — иконка нужна в навигации |
| Артефакт | `doc.text` | «документ стадии» — markdown и есть документ; `doc.badge.gearshape` для конфигов |
| Снапшот / diff | `plus.forwardslash.minus` | +/- — интернациональный знак диффа (GitHub-школа); для снапшота — `camera.on.rectangle` не брать (фото-метафора чужая), лучше `clock.arrow.circlepath` (состояние во времени) |
| Ревью | `text.magnifyingglass` | «читать пристально»; лупа одна = поиск, лупа с текстом = разбор текста |
| Комментарий | `text.bubble` | пузырь — вошедшая метафора; с бейджем количества — `text.bubble` + счётчик текстом, не рисовать число в иконке |
| Принять (approve) | `checkmark.circle.fill` | галка в круге, зелёная семантика ([06-color.md](06-color.md)); filled — это событие-состояние, не действие в покое |
| Вернуть на доработку (request changes) | `arrow.uturn.backward.circle` | разворот назад = цикл доработки; НЕ крестик (`xmark` = закрыть/отказ навсегда, у нас же — цикл) |
| Луп / итерация | `arrow.trianglehead.2.clockwise.rotate.90` (или `arrow.2.circlepath`) | круговые стрелки = повторение; счётчик итераций — цифра рядом, не внутри |
| Live-лог | `terminal` | моношрифт-консоль — донорская метафора Xcode/Terminal ([07-interaction.md](07-interaction.md) §5) |
| Стрим токенов | `waveform` | «живой поток»; поддерживает variable-color-анимацию (§9) |
| Аналитика | `chart.xyaxis.line` | линия на осях = графики лупов; `chart.bar` — если экран про счётчики |
| Claude / агент | **орб** (кастомный символ, §8) | фирменный элемент; SF-фолбэк — `circle.hexagongrid.circle` не брать, орб рисуем сами |
| Инбокс | `tray` / `tray.full` (есть непросмотренное) | системная метафора macOS Mail; пара empty/full — состояние формой |
| Настройки | `gearshape` | вошедшая в язык; без подписи можно |
| Запуск стадии | `play.circle` | универсальный «пуск»; filled на hover |
| Стоп/отмена работы агента | `stop.circle` | пара к play; не `xmark` (закрыть ≠ остановить) |

Правила таблицы: пары действий — парные символы (`play`/`stop`,
`tray`/`tray.full`); отвергнутые альтернативы фиксируем, чтобы не пересматривать.

## 4. Пиксельная сетка и оптика

- **Оптическая компенсация форм**: круг и треугольник, вписанные в тот же
  квадрат, кажутся меньше квадрата — поэтому в едином ряду круг рисуется чуть
  больше (перехлёст за сетку ~4–6% диаметра). SF Symbols делает это сам;
  правило нужно для кастомных иконок (§8) и для смешанных рядов «SF + кастом».
- **Выравнивание стрелок и шевронов**: шеврон в строке-раскрытии стоит по
  оптическому центру, не по геометрическому (визуальная масса шеврона смещена
  к острию). Проверка — отойти/прищуриться, а не мерить пиксели.
- **Целые пиксели**: иконки рендерить в целых px и на пиксельной сетке; иконка
  16 px, поставленная на координату 0.5, мылится — в тёмной теме мыло светлых
  линий на чёрном заметнее, чем в светлой.
- **Единая стилистика ряда**: один вес линии (наследуем от текста, §2.1), одни
  скругления, один принцип заполнения. В одном ряду не смешивать outline и
  filled как «просто разные иконки» — заполненность у нас означает состояние
  (§2.3), и только его.

## 5. Размеры

Единая шкала (пойдёт в [13-tokens.md](13-tokens.md)):

| Контекст | Кегль символа (pt) | Масштаб SF | Хит-зона |
|---|---|---|---|
| Пункт меню-статус, инлайн в тексте | = кегль текста (13) | small/medium | — (не кнопка) |
| Строка списка/инбокса | 13–14 | medium | вся строка |
| Сайдбар | 14–15 | medium | вся строка |
| Тулбар | 16–17 | medium | 28×28 px минимум ([07-interaction.md](07-interaction.md) §1.2) |
| Бейдж/чип (тип комментария, стадия на карточке) | 10–11 | small | чип целиком |
| Пустые состояния, онбординг | 34–48 | large | — |
| Статус-точка стадии | глиф 8–10 | — | строка/карточка |

**Императив: размеров — не больше этой шкалы.** Промежуточные «на глазок»
размеры плодят разнобой; нужен другой размер — меняется шкала, не экран.

## 6. Иконки-статусы стадий пайплайна

Статус — самое частое применение иконок у нас: колонка канбана, карточка,
инбокс, тулбар ревью.

- **Форма дублирует цвет** — обязательный императив: 8% мужчин различают
  красный/зелёный плохо; статус, различимый только цветом, для них пуст.
  Каждому статусу — свой глиф И свой цвет ([06-color.md](06-color.md)):

| Статус стадии | Глиф | Цвет |
|---|---|---|
| Не начата | `circle.dotted` | нейтральный (text.disabled, белый 38%) |
| В работе (Claude работает) | орб-индикатор / `circle.lefthalf.filled` + пульс | циан `#58C7FF` |
| Ждёт ревью | `hourglass` или `circle.fill` с точкой-бейджем | ультрамарин `#2F5CFF` |
| Принята (approved) | `checkmark.circle.fill` | sem.success (зелёный) |
| На доработке (request changes) | `arrow.uturn.backward.circle.fill` | sem.warning (янтарный) |
| Ошибка/блок | `exclamationmark.triangle.fill` | sem.error (красный) |

- Точки-статусы (8–10 px) допустимы в плотных местах (мини-карта стадий на
  карточке) — но тогда рядом текст или тултип раскрывает значение;
- статусные глифы — filled (это состояния, они «горят»), действия вокруг —
  outline: заполненность разводит «что есть» и «что можно сделать»;
- **исключение — сказать словами** (Бирман): когда система значков не
  вмещает особый случай статуса, не наращивай нотацию новым вариантом глифа —
  напиши слово или примечание у самого элемента. Одноразовое исключение не
  стоит нового элемента языка, который придётся выучить.

## 7. Тёмная тема: цвет иконок

Иконка подчиняется тем же уровням, что текст ([06-color.md](06-color.md);
значения токенов — [13-tokens.md](13-tokens.md)):

- **альфа-белые, не чисто-белые**: иконка в покое = цвет вторичного текста
  (text.secondary, белый 70% альфы); `#FFFFFF` 100% на `#07060B` — звенит и
  спорит с контентом;
- активная/выбранная — text.accent `#7C9AFF` (или белый там, где акцент
  занят фоном выделения); `#2F5CFF` — цвет заливок и фонов выделения, не
  глифов;
- disabled — 38% альфы (text.disabled), как disabled-текст;
- **никаких чёрных glyph-заливок**: тёмный глиф на тёмном фоне исчезает, а
  чёрная заливка внутри светлого контура даёт «дырку» — слои hierarchical
  режима решают это притуханием, не чернотой;
- семантические цвета — только у статусов (§6), не у декоративных и не у
  навигационных иконок: цветная иконка = сообщение о состоянии.

## 8. Кастомные иконки

### 8.1 Когда SF Symbols не хватает

Только два легальных повода: (1) доменная метафора, которой нет и не собрать
из вариантов (наш орб); (2) фирменный знак. Всё остальное — искать в
каталоге SF (6000+ символов, включая варианты `.fill`, `.badge.*`,
`.circle`, `.square`) прежде чем рисовать.

### 8.2 Как рисовать кастомный символ в стиле SF

- взять **шаблон Apple** (экспорт любого близкого символа из приложения
  SF Symbols → SVG-шаблон с направляющими) и рисовать в его слотах;
- обязателен вес **Regular** (минимум); лучше — Regular + Semibold + Bold,
  остальные веса система интерполирует (variable template);
- соблюдать оптические поля шаблона: символ должен вставать в текст без
  ручных отступов, как родной;
- толщина штриха и скругления — снятые с соседнего родного символа того же
  кегля, не «на глаз»;
- перечёркивание пиктограмм — диагональю сверху-слева вниз-вправо (ISO 3864,
  совет бюро); в SF Symbols брать готовые `.slash`-варианты, свою диагональ
  не рисовать;
- проверка качества: поставить кастомный символ в строку между двумя родными —
  если он выделяется весом, размером или позицией на базовой линии, дорабатывать.

## 9. Орб как фирменный элемент

Орб — сфера-«точка света» — наш знак Claude/агента
(визуальное направление серии: свет в темноте, концентрические круги).

- **Иконка приложения** — главный носитель орба: сфера с внутренним свечением
  ультрамарин→пурпур на почти чёрном; по гайдам macOS — в скруглённом квадрате
  системной маски.
- **Где орб уместен внутри UI**: индикатор «Claude работает» — в тулбаре
  change'а, на карточке в работе, в шапке live-лога. Это единственный
  анимированный элемент экрана (§10): пульс/переливы, пока идёт стрим.
- **Где орб неуместен**: как замена любой «AI-иконке» по всему приложению —
  орб в каждой строке превращает фирменный знак в обои и убивает его
  сигнальную ценность (это статус «агент работает», а не логотип-стикер).
  Не ставить орб: в пунктах меню, в пустых состояниях без связи с агентом,
  рядом с каждым сообщением лога.
- Статичный орб (без анимации) = «агент подключён, простаивает»; тонкая
  разница покоя и работы — анимацией, не сменой формы.

## 10. Анимация иконок

SF Symbols поддерживает символ-эффекты; используем скупо:

| Эффект | Когда | Пример у нас |
|---|---|---|
| variable color (перелив слоёв) | непрерывный live-процесс | `waveform` в шапке стрима токенов; орб при работе Claude |
| pulse | «жду/работаю» без прогресса | статус стадии «в работе» на карточке |
| bounce | однократное подтверждение события | галка approve в момент нажатия; bounce один раз, не циклом |
| replace (смена символа с анимацией) | переключение состояния | play → stop у запуска стадии |

Императивы: одновременно на экране **не больше одного** циклически
анимированного элемента (обычно — орб); анимация иконки всегда означает
процесс, никогда — «привлеки внимание к кнопке»; уважать Reduce Motion —
эффекты выключаются, состояние остаётся различимым статикой (форма+цвет, §6).

## Чек-лист

- [ ] У каждой иконки есть ответ, что она ускоряет; декоративные удалены
- [ ] Без подписи — только вошедшие в язык метафоры; у всех иконок-кнопок есть тултипы (с шорткатом)
- [ ] Меню — текстом; тулбар — SF Symbol + тултип; кнопки контента — текст (± иконка)
- [ ] Все иконки — SF Symbols; кегль и вес символа = кеглю и весу соседнего текста; baseline не подогнан вручную
- [ ] Рендер: monochrome/hierarchical; multicolor не используется
