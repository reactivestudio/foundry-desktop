# 14 · Сводный чек-лист экрана

> Серия design · [оглавление](README.md)

Финальная проверка любого макета/экрана перед тем, как считать его готовым.
Проходится сверху вниз, честно, по пунктам; «почти» = «нет». Подробные
чек-листы по темам — в конце каждой главы серии; этот — сжатая выжимка
для регулярного прогона. Порядок неслучаен: сначала смысл, потом форма.

## 0. Данные ([02](02-composition.md) §12)

- [ ] Макет собран на реальных данных, не на lorem и удобных коротких строках.
- [ ] Проверены экстремумы: самое длинное имя change/проекта; 0 / 1 / 999+ элементов; пустая колонка и колонка на 40 карточек; комментарий на 3000 знаков.
- [ ] Проверены деградированные состояния: foundry CLI недоступен, стрим оборвался, снапшот устарел во время ревью.

## 1. Польза и УВЛ ([01](01-principles.md))

- [ ] Названа главная задача экрана одним предложением; главное действие очевидно за 3 секунды.
- [ ] Для каждого элемента есть ответ «что сломается, если убрать». Нечего ответить — элемент убран.
- [ ] Прошёл УВЛ-итерацию: убрано всё, что дублирует, украшает или «на всякий случай». Потом ещё одну.
- [ ] Нет элементов ради симметрии/заполнения пустоты; пустота оставлена пустотой.
- [ ] Состояние системы видно без действий пользователя (что бежит, что ждёт, что упало).
- [ ] Деструктивное — обратимо (undo), а не «Вы уверены?».

## 2. Композиция ([02](02-composition.md))

- [ ] На экране 1–3 якорных объекта; тест прищуривания даёт правильный порядок чтения.
- [ ] Число вертикальных осей выравнивания минимально; каждая кромка выровнена с чем-то.
- [ ] Группировка читается воздухом и близостью, без рамок и линеек.
- [ ] При ресайзе окна тянется контент, а не хром; минимальный размер окна не ломает вёрстку.

## 3. Отступы ([03](03-spacing.md))

- [ ] Все отступы из шкалы токенов; произвольных значений нет.
- [ ] Правило внутреннего и внешнего: внутри элемента ≤ снаружи; внутри группы < между группами.
- [ ] Заголовок ближе к своему контенту, чем к предыдущему блоку (≈2:1).
- [ ] Одинаковые по смыслу блоки на разных экранах имеют одинаковые отступы.

## 4. Типографика и текст ([04](04-typography.md), [05](05-text.md))

- [ ] Кегли и веса только из типографической шкалы; уровней иерархии ≤3 на зону.
- [ ] Длина строки текста 45–75 символов; выключка флаговая, влево.
- [ ] Цифры в таблицах/счётчиках/логе — tabular; единицы вынесены из ячеек.
- [ ] Набор: «ёлочки», длинное тире, неразрывные пробелы (числа с единицами, № , инициалы).
- [ ] Кнопки — глаголы последствия, не «ОК/Да»; ошибки: что случилось + что делать.
- [ ] Пустые состояния подсказывают следующее действие, а не констатируют пустоту.
- [ ] Термины из словаря проекта; никакого «успешно», «пожалуйста», «Ой!».

## 5. Цвет ([06](06-color.md), [13](13-tokens.md))

- [ ] Все цвета — токены; фоны из шкалы поверхностей, слои не перепрыгивают.
- [ ] Один цветовой акцент на зону; маджента/градиент — не чаще одного на экран.
- [ ] Текст проходит контраст: primary/secondary/tertiary — альфа-белый; #2F5CFF текстом не используется.
- [ ] Семантический цвет = семантический смысл, всегда продублирован формой/иконкой.
- [ ] Свечение — максимум одно в поле зрения, никогда на данных.

## 6. Взаимодействие ([07](07-interaction.md))

- [ ] Кликабельные цели ≥24pt; действия рядом с объектом действия (Фиттс).
- [ ] Списки выбора сгруппированы; >10–15 пунктов — поиск (Хик).
- [ ] Отклик <100 мс визуально; >1 с — прогресс; оптимистичный UI где безопасно.
- [ ] Esc закрывает, Enter подтверждает, ⌘Z отменяет; фокус-ринг виден; tab-порядок осмыслен.
- [ ] Каждое действие доступно и мышью, и клавиатурой; hover — не единственный путь.
- [ ] Паттерны совпадают с привычками из GitHub/Xcode/Terminal (закон Якоба).

## 7. Контролы и поверхности ([08](08-controls.md), [09](09-surfaces.md))

- [ ] Primary-кнопка одна на зону; пары кнопок: действие справа, именованные лейблы.
- [ ] У контролов есть все состояния: hover, pressed, focus, disabled, loading — и они из токенов.
- [ ] Модалка оправдана (короткое обязательное решение); иначе — inline/инспектор/popover.
- [ ] Плашки/бейджи: альфа-заливка + цветной текст, кегль caption; не кричат.
- [ ] Разделители: сначала воздух; линейка — только осознанно, `border.subtle`.
- [ ] Тосты не сообщают очевидное, живут 4–6 с, несут Undo.
- [ ] Долгие события (конец/фейл стадии) зовут пользователя и вне окна: системная нотификация, бейдж дока ([09](09-surfaces.md)).

## 8. Таблицы и данные ([10](10-tables.md), [11](11-dataviz.md))

- [ ] Числа вправо, текст влево, заголовок = выравнивание колонки; зебры нет.
- [ ] Данных-чернил максимум: без 3D, теней, лишних сеток и легенд (Тафти).
- [ ] Тип графика отвечает вопросу; мало точек — таблица вместо графика.
- [ ] Подписи у данных, а не в легенде; оси от нуля для баров; lie factor = 1.
- [ ] Серий ≤4–5, дальше — small multiples; цвет серии стабилен между экранами.

## 9. Иконки ([12](12-icons.md))

- [ ] SF Symbols; вес = весу текста; иконка без подписи — только тултип + устоявшаяся метафора.
- [ ] Статусы: форма дублирует цвет; стилистика (вес, fill/outline) едина по экрану.
- [ ] Орб — только для AI-состояний, не размножается по экрану.

## 10. Финал

- [ ] Экран проверен на минимальной и максимальной яркости дисплея.
- [ ] Проверен ресайз до минимума окна и полноэкранный режим.
- [ ] Increase Contrast / Reduce Transparency / Reduce Motion не ломают экран.
- [ ] Прогнан симулятор дейтеранопии (дифф, статусы стадий).
- [ ] У иконок-кнопок есть accessibility-label (бесплатно из тултипов); всё достижимо с клавиатуры.
- [ ] Экспорт наружу (печать, PDF, копия диффа) — в светлой печатной палитре, не тёмный скриншот ([06](06-color.md)).
- [ ] Последний проход УВЛ: что ещё можно убрать? Убрать.
