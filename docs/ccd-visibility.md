# Видимость ранов foundry-desktop в Claude Code Desktop

Дата: 2026-07-14 · Статус: механизм найден и проверен экспериментом.

## Требование

Пользователь запускает ран в foundry-desktop → идёт в Claude Code Desktop
(CCD) и видит там эту сессию в конкретном проекте: что происходит, как агент
думает; после завершения — итог виден и в foundry-desktop.

## Почему «просто писать файлы» не работает

- `claude -p` пишет транскрипт в `~/.claude/projects/<путь>/<id>.jsonl` —
  это общее хранилище Claude Code, но **не** источник списка сессий CCD.
- Список CCD строится из его собственной метабазы
  `~/Library/Application Support/Claude/claude-code-sessions/<device>/<org>/local_*.json`;
  записи создаёт только сам CCD. Документация подтверждает: сессии из
  `claude -p`/SDK в session picker не появляются
  ([sessions](https://code.claude.com/docs/en/sessions.md)).
- Подделка записи метабазы отклонена: формат внутренний, включает
  runtime-состояние (конфиги MCP с полными схемами, VM-процесс и т.д.).

## Найденный механизм: deep link `claude://resume`

Реверс `Claude.app` (v1.20186.9, app.asar) показал недокументированный роут:

```
claude://resume?session=<cli-session-id (UUID)>
```

Обработчик в CCD: валидация UUID → `LocalSessions.importCliSession(id)` →
создание записи `local_<id>.json` в метабазе → открытие сессии в UI.
Импорт читает транскрипт из `~/.claude/projects` (ошибка «transcript may
have been removed», если файла нет). Строки ошибок в бандле прямо описывают
сценарий «deep link references a CLI session» — то есть импорт CLI-сессий
задуман, просто не документирован.

**Эксперимент (2026-07-14, подтверждено):** `open
"claude://resume?session=dbae5a6c-…"` для завершённой сессии демо-прогона →
в метабазе появилась запись `local_dbae5a6c-…` (`cliSessionId` = наш id,
`cwd` = каталог проекта), сессия видна в списке CCD.

## Целевая петля foundry-desktop

1. Ран стартует: `claude -p … --output-format stream-json`; из события
   `system/init` берём `session_id`.
2. Сразу же (опция «Показывать в Claude Desktop», по умолчанию вкл.) —
   `open claude://resume?session=<id>`: CCD импортирует сессию и открывает
   её; пользователь смотрит ход работы там.
3. foundry-desktop параллельно ведёт свою live-ленту и по `result` показывает
   итог. Транскрипт один и тот же — файл в `~/.claude/projects`.

## Риски и открытые вопросы

- **Недокументированный роут**: может измениться в любом релизе CCD.
  Митигация: изолировать в одном типе (`ClaudeDesktopLink`), деградация
  мягкая (сессия просто не откроется в CCD; наш просмотр не страдает).
- **Live-обновление импортированной сессии во время рана** — проверяется
  практикой: импорт в момент старта гарантированно подхватывает транскрипт,
  но частота обновления UI CCD для дописываемого извне файла не контрактная.
  Если окажется статичным — уровень отката: повторный импорт по завершении
  (итог виден всегда).
- Deep link уводит фокус в CCD — поэтому опция, а не безусловное поведение.
