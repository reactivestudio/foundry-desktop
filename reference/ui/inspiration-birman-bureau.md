# Референс: приёмы из проектов Бирмана и бюро Горбунова

Мудборд-конспект переносимых дизайн-приёмов для foundry-desktop (тёмная тема,
десктопная плотность). Собран из реальных проектов, не из общих советов. Каждый
приём: **что делают → где у нас → что меняет**. Применённое в макетах помечено ✅,
заложенное в бэклог — ⏳.

Экраны foundry: сайдбар (проекты) + кросс-проектный инбокс ревью · канбан
change'ей по 8 стадиям CRISPY · ревью диффа с типизированными комментариями ·
live-лог · аналитика лупов.

---

## Илья Бирман

Главный источник — **Superpedestrian** (пульт управления флотом самокатов):
плотный оперативный интерфейс, ближайший по духу к нашему пульту.

### Композиция / панели
- ⏳ **Виджет-сводка = контрол-панель** (Superpedestrian). Счётчик стадии
  одновременно показывает число, служит легендой и фильтрует правую область. →
  Счётчики в сайдбаре/шапке инбокса кликабельны и фильтруют список. Убирает
  отдельную панель фильтров.
- ⏳ **Секции по приоритету задач, не по алфавиту** (Superpedestrian: Deployment →
  Pickup → Risk → Checkup → Raw). → Инбокс: «Требует решения → Падения CI →
  Долгоживущие → Оптимизация → Сырой лог». Критичное всегда над сгибом.
- ✅ **Панели приклеены к краям окна**, никаких плавающих сайдбаров с внешним
  зазором (заметка «Никогда не отрывайте панели от краёв»; Figma 3.0/macOS
  откатывали). Край — бесконечная мишень по Фиттсу. → Сайдбар и панель
  комментариев у кромок.
- ⏳ **Раскрываемый доступ к сырым данным** стрелкой внизу секции
  (Superpedestrian: Raw Data Access). → Под секциями инбокса/аналитики —
  свёрнутый «Открыть в логе / сырой запрос».

### Данные / таблицы
- ✅ **Микро-таблица с инлайн-кнопкой вероятной команды** (Superpedestrian:
  id+статус+гео+quick-action). → Секция «Требует внимания»: строка = change +
  инлайн Approve/Retry/Open. Потоковое ревью без захода в карточку.
- ⏳ **Гистограмма как орган пакетного выбора** (Superpedestrian: заряд → «выбрать
  все < 37%» → задание). → Аналитика: протяжка по диапазону гистограммы →
  массовое действие (retry/переназначить). Спарклайны из пассивных в управляющие.
- ⏳ **Исторический базлайн рядом с числом** (сегодня/3д/10д). → У каждой метрики
  фон-базлайн: «5 failed» читается как норма/тревога, а не голое число.

### Модальность / тексты
- ✅ **Немодальность типизированных комментариев**, тип задаётся инлайн, без
  режима-модалки; ноль обязательных полей. → Комментарий в диффе ставится в
  несколько мест подряд, не «проваливаясь» в диалог.
- ✅ **Hover is over**: важные действия не под наведением, всегда видимы. →
  Действия карточки/комментария постоянны.
- ⏳ **Инфостиль + запрет отрицаний** в лейблах: «Показывать пройденные», глаголы
  вместо канцелярита. → Ревизия всех строк стадий/фильтров/ошибок.
- ⏳ **Логика структуры важнее сценария** (архив вакансий — не подпункт создания).
  → «История change'ей» — самостоятельный раздел; инбокс — верхнеуровневый.

## Бюро Горбунова

Изучены fxpro-markets, admiral-markets, birdview, action-otchety,
action-dogovory, jurist-laws, worldchess-broadcasts, onetwotrip, glavbukh.

### Композиция / якорные объекты
- ⏳ **Незыблемые якоря между сценами** (Worldchess): статус/таймер/орб в одних
  координатах при переходах инбокс→канбан→дифф→лог. Глаз не переискивает якорь.
- ⏳ **Активные объекты — в закреплённой полосе** (FXPro, Admiral): in-flight
  агенты в отдельной pinned-панели, не вперемешку с очередью.
- ⏳ **Семантический зум портфель→элемент без модалок** (Birdview): инбокс
  bird's-eye по проектам, зум растит плотность вплоть до одного change'а.
- ⏳ **Dual-panel «исходник + чеклист рисков»** (Action-Договоры): рядом с диффом
  колонка распознанных пунктов/чеклист CRISPY — ревью как направленное сканирование.

### Плотность / таблицы
- ✅ **Подсветка только 2 опорных записей** (Onetwotrip: самый быстрый+дешёвый). →
  Инбокс маркирует «самый старый на ревью» и «самый рискованный», остальное
  нейтрально. Решение в один взгляд.
- ⏳ **Скобочный коннектор + ховер-подсветка связанных строк** (FXPro). →
  Связанные строки диффа/коммиты одного change'а — скоба слева, ховер
  подсвечивает всю группу.
- ✅ **Трёхстатусная иконочная система** (Action: принято/уточнить/отклонено ✓✗).
  → Статус ревью и каждой стадии — фиксированный значок, скан без чтения.
- ⏳ **Инлайн-загрузка + графическая метафора состояния в строке** (Birdview:
  волнистая = отпуск). → Список агентов/стадий: занятость и деградация в строке.

### Данные / цвет
- ✅ **Цвет только на значимом изменении, база нейтральна** (FXPro). → Общий
  принцип тёмной темы против визуальной паники.
- ⏳ **Факт вместо прогноза** (Birdview): показывать реальные итерации/откаты, не
  предсказание. Убирает ложную уверенность в авто-оценке.
- ✅ **Типизированные риск-маркеры комментариев** (Action-Договоры:
  блокер/баг/вопрос/nit) — тип читается цветом до текста.
- ⏳ **Мнемо-имена вместо хешей** (Admiral: Рублёвый вместо номера счёта). →
  change'ам — узнаваемые имена/эмодзи-якорь.
- ⏳ **Параллельная лента второстепенных событий** (Worldchess). → Live-лог сбоку
  как непрерывная лента, не перебивая фокус.

### Микродетали / Фиттс
- ⏳ **Always-active фильтр без фокуса на поле** (FXPro): печать где угодно
  фильтрует. Минус клик, минус промах по узкому полю.
- ✅ **Прогресс как заливка самого объекта** (Admiral: имя закрашивается по
  анкете). → 8 стадий CRISPY закрашивают орб/кольцо статуса change'а, а не
  отдельный бар. Плотнее и на бренде (орб-центричность).
- ⏳ **Drag файла прямо в окно** (Action-Отчёты): drop патча/лога в change, не
  диалог.
- ⏳ **Контекстные кросс-ссылки** (Jurist-Laws): в диффе — инлайн-ссылки на
  связанные change'и и пункт правила CRISPY.
- ✅ **Пиктограмма-якорь в начале строки** (Glavbukh): тип события опознаётся
  периферийным зрением в инбоксе/логе.

## Приоритет внедрения (сведено, дедуплицировано)

**Применено в макетах ревизией:** инлайн-действия в строках внимания ·
прогресс-кольцо стадий на статусе change'а · типизированные маркеры комментариев ·
подсветка только опорных строк · нейтральная база + цвет на значимом · панели у
краёв · пиктограммы-якоря · немодальные комментарии · командная палитра ⌘K ·
тост с Undo · **флоу-карта пайплайна графом (§9)** · **экран настроек с пресетами (§10)**.

**Внедрено в ревизии инбокса (§1, «дизайн-детали»):** мнемо-имена вместо хешей
(Admiral) · позиция в CRISPY словами «стадия N из 8», графический прогресс не
дублируется в строке — только степпер инспектора (Admiral заливка объекта + УВЛ:
сегментный рейл в строке убран как лишний) · секции
по приоритету, не по проекту (Superpedestrian) · инфостиль-заголовки-глаголы
(Бирман) · виджет-сводка как фильтр — счётчик и есть пульт (Superpedestrian) ·
исторический базлайн у числа «3 · норма 2» (Бирман) · две опорные строки «дольше
всех»/«крупный дифф» (Onetwotrip) · закреплённая полоса in-flight у нижней кромки
(FXPro/Admiral) · always-active фильтр-подсказка (FXPro).

**Внедрено в проработке всех экранов (v8):** dual-panel «дифф + чеклист рисков
CRISPY» с трёхзначковой системой ✓▲✗○ (Action-Договоры, §3) · скобочный коннектор
связанных ханков с ховер-группой и подписью (FXPro, §3) · кросс-ссылки на
change-источник и пункт правила (Jurist-Laws, §3) · график истории прогонов как
орган пакетного выбора — протяжка по диапазону → массовое действие (Superpedestrian,
§4) · карты канбана и мини-строки мульти-окна в анатомии инбокса, чип стадии не
дублируется = колонка (УВЛ, §6/§7).

**Бэклог (документирован, не нарисован):** незыблемые якоря между сценами ·
семантический зум портфель→элемент · факт вместо прогноза · drag-drop файла в change.
