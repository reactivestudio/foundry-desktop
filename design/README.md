# foundry-desktop — иконка приложения

Иконка «Восход»: тёмная сфера, встающая из-за нижнего края, освещённая источником
за верхне-правым лимбом. Метафора — control plane, который «пробуждается» над
пайплайном. Референс — рассветная сфера Титана (`reference-sunrise-sphere.png`),
адаптированная в фирменную палитру.

## Конвейер

```
app-icon.html  →  foundry-app-icon-1024.png  →  make-appiconset.py  →  App/Assets.xcassets/AppIcon.appiconset/
  генератор            мастер-рендер                  нарезка              единственный источник правды
```

- `app-icon.html` — самодостаточный генератор (Canvas 2D). Открыть в браузере,
  кнопка «Скачать PNG 1024×1024» экспортирует артворк.
- `foundry-app-icon-1024.png` — мастер-рендер 1024×1024: квадрат, без скругления,
  непрозрачный. Единственный вход нарезки.
- `make-appiconset.py` — нарезка в набор для Xcode (нужен Pillow). Мелкие размеры
  супер-семплированы для гладких углов.
- `reference-sunrise-sphere.png` — исходный референс для сверки.

Готового набора рядом с мастером **не держим**. Держали — и он разошёлся с тем,
что реально шипится: артворк правили здесь, а приложение продолжало собираться со
старой иконкой (см. историю: правки «толще серп» и «моушен-блюр» доехали до
`design/`, но не до `App/`). Набор существует ровно в одном месте — там, откуда
его берёт сборка.

## Перегенерация

После правок в `app-icon.html`:

```sh
open design/app-icon.html                                     # 1. скачать PNG 1024
mv ~/Downloads/foundry-app-icon-1024.png design/foundry-app-icon-1024.png   # 2. заменить мастер
python3 design/make-appiconset.py                             # 3. нарезать набор
```

Правка артворка **не закончена**, пока не прогнан шаг 3.

## Сетка macOS

Артворк кладётся не в край плитки, а по сетке Big Sur: на канвасе 1024 тело
занимает центральные **824×824** (прозрачные поля по 100 px), скругление r≈184.
Без полей иконка в Dock выглядит заметно крупнее системных — у macOS поля входят
в саму картинку, а не добавляются системой.

## Палитра (webphone-токены)

| Токен | HEX | Роль |
|-------|-----|------|
| Void   | `#050510` | фон, дно хода |
| Navy   | `#1B1690` | глубокая тень |
| Violet | `#4522CE` | пурпур в центре тела |
| Indigo | `#653AF4` | сине-пурпурный акцент |
| Sky    | `#38AFF5` | освещённая кромка |
| Halo   | `#161A2C` | атмосферная дымка над лимбом |

Ход тела (дизеринг убирает бандинг): освещённая кромка — Sky, центр тела уведён
в сине-пурпур/фиолет, к тени — Navy → Void:
`Sky #3AA6F4 → #4A63F7 → #653AF4 → #5A2CDF → #4522CE → Navy #221286 → Void #050510`.
Пурпур живёт именно в центре тела (насыщенный, но не перетягивает); кромка остаётся
холодной Sky. Ранняя «чисто синяя» версия без пурпура заменена на этот ход.

## Приёмы рендера (зафиксированы)

- **Свет за кромкой.** Радиальный градиент тела с центром за верхне-правым лимбом
  (`895,405`, радиус `1180` при 1024) — свет «встаёт» из-за края, большой мягкий
  переход по всему телу, без пятна в центре.
- **Прогрессивный односторонний блюр.** Правая половина и центр — резко; влево от
  центра размытие плавно нарастает пятью слоями (`1.2 → 5 → 12 → 30 → 92 px`),
  сведёнными широкими горизонтальными масками без ступеней. Левый лимб тает в фон.
- **Световой серп.** Только тонкое чёткое ядро на правой кромке (без широкого
  свечения), яркость затухает колоколом к концам дуги (пик `−38°`).
- **Фон — радиальная дымка**, не линейная: линейный градиент с плато давал
  горизонтальную полосу-ступень, радиальная дымка над лимбом её убирает.
- **Дизеринг TPDF** (`±7.5`) поверх всего — против бандинга в тёмных переходах.
